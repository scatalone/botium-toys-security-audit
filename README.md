# Botium Toys Cybersecurity Audit

## Overview
This project simulates an internal cybersecurity audit for Botium Toys, a fictional company expanding its online toy sales globally. The audit was conducted using the NIST Cybersecurity Framework (CSF) to evaluate the organization's security posture.

## Objectives
- Identify security risks and vulnerabilities
- Review existing technical, administrative, and physical controls
- Evaluate compliance with PCI DSS and GDPR
- Provide recommendations to improve security posture

## Framework Used
NIST Cybersecurity Framework (CSF)

Core functions:
- Identify
- Protect
- Detect
- Respond
- Recover
- Govern

## Key Findings
Several major control gaps were identified including:

- Lack of least privilege access controls
- No intrusion detection system
- No encryption for sensitive data
- Weak password policies
- No disaster recovery plan
- Missing backups of critical systems

These weaknesses increase the risk of data breaches and regulatory penalties.

## Compliance Risks
The organization is currently **non-compliant** with:

- PCI DSS
- GDPR
- SOC security best practices

## Recommendations

1. Implement least privilege access controls
2. Deploy intrusion detection systems
3. Encrypt sensitive customer data
4. Strengthen password policies
5. Implement a disaster recovery plan
6. Introduce a password management system
7. Classify and inventory sensitive data

- Security auditing
- Risk assessment
